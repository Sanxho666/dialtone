--- layout: page title: Installing Dialtone Locally description: A more thorough guide for installing Dialtone locally. ---

First things first: we're excited you want to install Dialtone locally as this most likely means you'll be contributing soon! Before you get to get started though, please make sure you've read our contributing docs.

The below instructions assumes a brand new build. If you've installed Ruby, Node and NPM for other projects, please skip ahead.

Install Ruby via Homebrew

In order to run Jekyll, you will need a full Ruby development environment. If you already have a Ruby environment installed, move on to the next step.

If you don't have Ruby installed, it's recommended you use Homebrew to install Ruby. First you must install Homebrew. Enter the following command from a Terminal window:

{% highlight ruby linenos %} /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)" {% endhighlight %}

This will take a few minutes to install. Once Homebrew has installed, run the following commands to install Ruby:

{% highlight ruby linenos %} brew install ruby brew install brew-gem brew gem install bundler {% endhighlight %}

Install Node & NPM

To run Dialtone locally, you must have Node and NPM (Node Packet Manager) installed. Click here to download Node. The recommended Node version is fine. NPM is included with Node. If you already have Node installed, you may move onto the next step.

Once Node finishes installing, ensure it is installed properly by typing the following command in your Terminal window:

{% highlight ruby linenos %} node -v {% endhighlight %}

You should see the following response:

{% highlight ruby linenos %} v12.18.2 {% endhighlight %}

Clone project repo

Download the repo to your machine:

{% highlight ruby linenos %} // SSH git clone git@github.com:dialpad/dialtone.git // HTTPS git clone https://github.com/dialpad/dialtone.git {% endhighlight %}

Then cd into the Dialtone directory:

{% highlight ruby linenos %} cd ./path/to/dialtone {% endhighlight %}

Install Gulp

Dialtone uses Gulp to automate its various workflows.

{% highlight ruby linenos %} npm install --global gulp-cli {% endhighlight %}

Install Bundler & Jekyll

The Dialtone documentation website is built using Jekyll, a static HTML generator. To install Jekyll and its dependencies, do the following steps:

  1. Switch to the docs folder in the Dialtone directory. cd docs
  2. Install Bundler and Jekyll locally: gem install bundler jekyll
  3. Install the Bundle: bundle install

Install project dependencies

Navigate out of the docs folder to root project folder. Run the following command to install all other project dependencies:

{% highlight ruby linenos %} npm install {% endhighlight %}

Building Dialtone

You're now ready to build Dialtone! To build build, run the following command:

{% highlight ruby linenos %} gulp watch {% endhighlight %}

Once completed, view the doc site by visiting http://localhost:4000.

Dialtone provides the following commands:

  • gulp — This compiles Dialtone (CSS and Docs) only. It will not run the website locally.
  • gulp watch — This compiles Dialtone (CSS and Docs), serves the website locally, and dynamically rebuilds the site once any changes are saved.
  • gulp icons — This compiles icons only.
  • gulp version — This updates the version number on the documentation website.

Dialtone uses Browsersync to serve the compiled Jekyll website locally. This means you can have multiple browser windows open that sync actions together. This allows you to test media queries quickly from different window sizes.
