{% header "h2", "Adding Dialtone To Your Project" %}
{% paragraph %}To take advantage of Dialtone's customizations, classes, and variables in your project, you will want to install Dialtone via npm.{% endparagraph %} {% codeWell %} {% codeWellFooter %} {% highlight bash linenos %} npm install @dialpad/dialtone {% endhighlight %} {% endcodeWellFooter %} {% endcodeWell%}
{% paragraph %}Add the following line in your Less file:{% endparagraph %} {% codeWell %} {% codeWellFooter %} {% highlight less linenos %} @import "your/path/to/dialtone.less"; {% endhighlight %} {% endcodeWellFooter %} {% endcodeWell%}
{% paragraph %}If you only need access to Dialtone's variables and customizations to build a file and don't need the whole library exported, use this line instead in your Less file:{% endparagraph %} {% codeWell %} {% codeWellFooter %} {% highlight less linenos %} @import (reference) "your/path/to/dialtone.less"; {% endhighlight %} {% endcodeWellFooter %} {% endcodeWell%}
{% header "h2", "Theming" %}
{% paragraph %}To customize your Dialtone installation, create a new Less file in the {% code %}/lib/build/less/themes/{% endcode %} folder. By default this will compile during build, which can be added as an additional CSS file in your page {% code %}head{% endcode %}.{% endparagraph %} {% paragraph %}{% endparagraph %} {% codeWell %} {% codeWellFooter %} {% highlight less linenos %} @import "your/path/to/dialtone.less"; {% endhighlight %} {% endcodeWellFooter %} {% endcodeWell%}
{% paragraph %}If you only need access to Dialtone's variables and customizations to build a file and don't need the whole library exported, use this line instead in your Less file:{% endparagraph %} {% codeWell %} {% codeWellFooter %} {% highlight less linenos %} @import (reference) "your/path/to/dialtone.less"; {% endhighlight %} {% endcodeWellFooter %} {% endcodeWell%}
