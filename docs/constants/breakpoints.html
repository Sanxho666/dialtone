--- layout: page title: Breakpoints description: Most utility and component classes will have responsive classes. Using specific breakpoint constants, we create max-width media queries represented in conditional prefixes. These prefixed classes allow you to show apply a style or property within a specific breakpoint. ---

To help keep prefixes concise, we use abbreviations. This syntax is used consistently across all responsive classes. As the viewport size grows, you can change an elements properties. For example, you can set an element to display normally, but be hidden at smaller sizes: .d-block .sm:d-none.

Breakpoint Constant Prefix Media Query
Small $width-sm .sm:{class} max-width: 480px
Medium $width-md .md:{class} max-width: 768px
Large $width-lg .lg:{class} max-width: 1024px
Extra large $width-xl .xl:{class} max-width: 1280px

Note: These breakpoints may change.

Mixin

To generate set specific properties within media queries, use the media query mixin. The mixin works by passing one of the 4 breakpoints above (i.e. rwd(lg)).

{% highlight sass linenos %} .dds-style { display: none; @include rwd(md) { display: block; } } {% endhighlight %}

The resulting CSS would be:

{% highlight sass linenos %} .dds-style { display: none; @media (max-width: 768px) { display: block; } } {% endhighlight %}

Responsive Variants

To generate all responsive width variants for a property, use the responsive variant mixins like so:

{% highlight sass linenos %} @each $breakpoint, $variant in $rwd-variants { @include rwd($breakpoint) { .#{$variant}d-block { display: block; } } } {% endhighlight %}

The resulting CSS would be:

{% highlight sass linenos %} .d-block { display: block; } @media (max-width: 480px) { .sm:d-block { display: block; } } @media (max-width: 768px) { .md:d-block { display: block; } } @media (max-width: 1024px) { .lg:d-block { display: block; } } @media (max-width: 1280px) { .xl:d-block { display: block; } } {% endhighlight %}
