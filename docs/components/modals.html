--- layout: page title: Modals description: Modals are an dialog overlay that prevents the user from interacting with the rest of the page until an action is taken within the dialog or is dismissed. These dialogs are purposefully disruptive, interrupting the user's workflow. They should be used thoughtfully and carefully, specifically in moments where focus is required or an action must be taken. ---

Accessibility

On their own, modals are not very accessible as it visually brings forward content not in the document flow. To correct this, modals in Dialtone have been designed with accessibility in mind. When opened, navigation controls are constrained to elements within the modals.

To ensure maximum compatibility, all a tags must have an hrefattribute. Also any elements which you don't want to be focusable (but might be focusable by default) must have their tabindex set to -1.
